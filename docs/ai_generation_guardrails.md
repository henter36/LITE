# AI Generation Guardrails

**Project:** Marketing OS Lite  
**Version:** Phase 2 — AI Provider Layer  
**Last updated:** 2026-05-02

---

## Overview

All content generated by Marketing OS Lite — whether from the mock provider or the OpenAI provider — passes through a two-layer guardrail system before it is stored or returned to any client.

Human approval is always required before generated content can advance to "approved" status. The AI layer produces drafts only.

---

## Layer 1 — Hard-Banned Phrase Filter

Applies to every generation regardless of provider or brand profile configuration. These phrases (and any variation matched case-insensitively as whole words) are replaced with `[filtered]` in all output fields.

| Banned phrase |
|---------------|
| guaranteed sales |
| 100% guaranteed |
| guaranteed results |
| instant results guaranteed |
| risk-free profits |
| proven to double |
| proven to triple |
| overnight success |
| get rich |
| make money fast |
| no effort required |
| results guaranteed |
| money-back guaranteed |
| financial freedom guaranteed |

These rules cannot be disabled by any workspace configuration.

---

## Layer 2 — Workspace Forbidden Claims

Each workspace may define a `forbiddenClaims` field on its brand profile. This is a free-text field containing phrases separated by newlines, periods, commas, exclamation marks, question marks, or semicolons.

Each phrase is matched case-insensitively as a whole-word pattern. Matches are replaced with `[filtered]` in all output fields: `headline`, `shortCaption`, `longCaption`, `cta`, `hashtags`, `videoScript`, `storyboardOutline`.

**Example brand profile forbidden claims:**
```
We eliminate all risk
Scientifically proven to cure
Industry-leading guaranteed ROI
```

Any of these phrases appearing in generated output would become `[filtered]`.

---

## OpenAI System Prompt Guardrails

When the OpenAI provider is active, guardrails are also injected directly into the model's system prompt:

1. The full `forbiddenClaims` text is listed in a `FORBIDDEN CLAIMS` section of the system prompt.
2. Hard rules explicitly prohibit:
   - Unsupported performance promises
   - Fabricated statistics, testimonials, or social proof
   - Competitor comparisons by name
   - Implying content is approved or final
   - Content that violates major advertising platform policies

Post-generation filtering (Layers 1 and 2) still runs even when OpenAI is used, providing defence-in-depth: a guardrail that fires at both generation time and post-processing.

---

## What Guardrails Do NOT Cover

The guardrail system filters specific phrases. It does not:

- Review factual accuracy of claims about a business
- Verify product/service descriptions against real offerings
- Check legal compliance for specific industries (finance, healthcare, law)
- Prevent subtly misleading framing that doesn't match a banned phrase

**Human review and approval is required before any content is used.** The content studio approval workflow enforces this — no generated asset can be marked "approved" without explicit human action.

---

## Prompt Version

The current prompt version is `v1.0`, stored in the `prompt_version` column of `generated_assets`.

When the system prompt or output structure changes in a way that materially affects generation behaviour, the prompt version must be incremented (e.g., `v1.1`, `v2.0`) so historical assets can be traced to the exact prompt that generated them.

---

## Applying Guardrails to Channel Variants

Channel variants (per-platform overrides stored in `channel_variants`) inherit their `headline` and `caption` from the guardrail-filtered base output. Channel-specific CTA and hashtag overrides are inserted after filtering and do not contain brand copy, so no additional filtering is applied to them.

---

## Disabling Guardrails

Hard-banned phrase filtering cannot be disabled. Workspace `forbiddenClaims` can be cleared by an admin, which disables Layer 2 for that workspace.

## Future Considerations
