# Asset Library Foundation ŌĆö Slice 1 Report

## Objective
Create a governed asset library foundation for campaign creative assets before adding image/video generation.

## Completed work

### 1. Backend / Data model
- `lib/db/src/schema/media-assets.ts` ŌĆö extended `mediaAssetsTable` with two new columns:
  - `sourceType` (text, NOT NULL, default `external_url`): `uploaded | external_url | generated_later`
  - `usageRightsNotes` (text, NOT NULL, default `""`): free-text usage rights enforcement
- Columns added to the live database via `ALTER TABLE ... ADD COLUMN IF NOT EXISTS`.
- Existing fields retained: `workspaceId`, `campaignId` (optional), `title`, `type` (image/video/document/link/other), `description`, `channel`, `status` (draft/needs_review/approved/rejected), `createdBy`, `createdAt`, `updatedAt`.

### 2. Backend routes (`artifacts/api-server/src/routes/mediaAssets.ts`)
- `GET /api/media-assets?workspaceId=&campaignId=&status=` ŌĆö list assets; workspace isolation enforced
- `POST /api/media-assets` ŌĆö create asset reference; requires `editor` role or above
- `PATCH /api/media-assets/:id` ŌĆö update metadata/status; requires `editor` role
- `DELETE /api/media-assets/:id` ŌĆö delete asset reference; requires `admin` role; blocks deletion of approved assets
- Guardrail: `usageRightsNotes` required when status is `approved` (enforced on both create and update)
- Audit log written for create, update, and delete
- No binary upload, no AI generation, no auto-publishing

### 3. OpenAPI spec (`lib/api-spec/openapi.yaml`)
- Added `media-assets` tag
- Added paths: `GET/POST /media-assets`, `PATCH/DELETE /media-assets/{id}`
- Added schemas: `MediaAsset`, `CreateMediaAssetBody`, `UpdateMediaAssetBody`
- Codegen re-run cleanly (`NODE_OPTIONS=--max-old-space-size=8192`)

### 4. Generated client hooks (`lib/api-client-react`)
- `useListMediaAssets`, `useCreateMediaAsset`, `useUpdateMediaAsset`, `useDeleteMediaAsset`
- Generated by Orval; libs typecheck passed

### 5. Asset Library UI (`artifacts/marketing-os/src/pages/asset-library.tsx`)
- Sidebar nav item "Asset Library" at `/asset-library`
- Shows all assets for the active workspace
- Asset cards show: type icon, title, description, type badge, source badge, status badge, URL/reference, linked campaign name, usage rights notes
- Create asset reference form (dialog): title, type, sourceType, URL/reference, description, channel, status, campaign link, usage rights notes
- Edit dialog (same form pre-populated)
- Approve/reject quick-action buttons per asset
- Delete with confirmation dialog (blocks approved assets)
- Viewer read-only banner; all mutations disabled for viewers
- Client-side enforcement: cannot approve without usage rights notes

### 6. Campaign Detail integration (`artifacts/marketing-os/src/pages/campaign-detail.tsx`)
- New "Creative Assets" tab in campaign detail alongside Ad Content, Publish, Tracking Links
- Shows media assets linked to the campaign
- "Attach Asset" form: link an existing workspace asset to this campaign via PATCH
- Viewer-safe: read-only for viewers

### 7. Routing
- `artifacts/marketing-os/src/App.tsx` ŌĆö route `/asset-library` added
- `artifacts/marketing-os/src/components/layout/sidebar-layout.tsx` ŌĆö "Asset Library" nav item added
