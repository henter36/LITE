Build a lightweight MVP called Marketing OS Lite.

Objective:
Create a web platform that helps small businesses plan, generate, approve, and monitor marketing campaigns for Instagram, Snapchat, YouTube, and X/Twitter from one dashboard.

Important scope:
This is a Lite MVP for proof of concept.
Do NOT implement real ad spending.
Do NOT implement automatic budget changes.
Do NOT publish live ads automatically.
Do NOT connect real payment methods.
Do NOT store platform passwords.
Use simulated/mock ad platform integrations for now, but design the code so real APIs can be added later.

Tech stack:
- Frontend: React + TypeScript + Vite
- Backend: Python FastAPI
- Database: SQLite
- Styling: Tailwind CSS
- API documentation: OpenAPI from FastAPI
- Run frontend and backend in Replit
- Provide clear README with run instructions

Core modules:

1. Authentication-lite
Create simple local user/workspace handling without production-grade auth.
Tables:
- users
- workspaces
- workspace_members

2. Workspace
Each customer has a workspace.
Workspace fields:
- id
- name
- business_type
- country
- language
- default_currency
- created_at

3. Brand Profile
Allow the user to define:
- brand name
- tone of voice
- target audience
- products/services
- forbidden claims
- preferred channels
- visual notes

Tables:
- brand_profiles
- brand_rules

4. Campaign Planner
Allow the user to create a campaign with:
- campaign name
- objective: awareness, traffic, leads, sales
- product/service
- audience
- geography
- budget suggestion only
- start date
- end date
- channels: Instagram, Snapchat, YouTube, X
- landing URL or WhatsApp link

Important:
Budget is only advisory in Lite MVP. Do not execute spending.

Tables:
- campaigns
- campaign_channels

5. AI Content Studio - mock generation
Create a content generation screen that produces campaign assets.
Use deterministic mock generation for now, not paid AI APIs unless environment variables are configured.
Generate:
- ad headline
- short caption
- long caption
- CTA
- hashtag suggestions
- channel-specific variants
- simple video script
- storyboard outline

Tables:
- generated_assets
- channel_variants

- draft
- reviewed
- approved
- rejected

6. Approval Workflow
Before any campaign can be marked ready, require manual approval.

- approve campaign

10. Recommendations Engine - rules based
Create a simple rules-based recommendation engine.
Examples:
- If CTR is low, suggest improving hook/headline.
- If CPC is high, suggest narrowing audience or testing creative.
- If conversions are low but clicks are high, suggest landing page review.
- If one channel performs better, suggest increasing focus, but do not change budget automatically.

- recommendations
