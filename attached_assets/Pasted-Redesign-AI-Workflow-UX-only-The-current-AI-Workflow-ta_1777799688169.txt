Objective:
Replace the current long 6-step technical AI Workflow UI with a simple Arabic RTL “Campaign Launch Assistant” experience that is easy for non-technical users.

Current problem:
The existing workflow shows too many technical steps:
- Client Intake
- Strategy Brief
- Creative Brief
- Text Suggestions
- Image Prompt Specs
- Video Script Specs

This is too developer-oriented and hard for users.

New UX concept:
Create one simplified guided assistant called:
"مساعد إطلاق الحملة"

The user should understand:
- ماذا يحتاج النظام؟
- ما الخطوة الحالية؟
- ما الذي سيتم توليده؟
- ما الذي يحتاج مراجعته؟
- أن كل شيء مسودة فقط
- أنه لا يوجد توليد صور أو فيديو أو نشر مباشر

Required redesign:

1. Top assistant summary card

At the top of the AI Workflow tab, show a premium card:

Title:
"مساعد إطلاق الحملة"

Subtitle:
"حوّل معلومات الحملة إلى استراتيجية، نصوص، ومواصفات إبداعية جاهزة للمراجعة."

Show:
- current campaign name if available
- workflow progress percentage or completed step count
- next recommended step
- draft-only badge
- safe note: "لا يتم الاعتماد أو النشر تلقائيًا"

Use emerald/teal styling, soft white card, RTL layout.

2. Replace 6 exposed technical cards with 4 user-friendly stages

Use four stages only:

Stage 1:
"فهم الحملة"
Combines client intake and missing context.

Stage 2:
"بناء الاستراتيجية"
Covers strategy brief.

Stage 3:
"تجهيز المحتوى"
Covers text suggestions, hooks, captions, CTA, ad copy.

Stage 4:
"المواصفات الإبداعية"
Covers image prompt specs and video script/storyboard specs.
Clearly state:
- "مواصفات الصورة نصية فقط ولا يتم توليد صور."
- "نص الفيديو ولوحة المشاهد تخطيط نصي فقط ولا يتم توليد فيديو أو رفع ملفات."

3. Active-step layout

Do not show all detailed forms as long stacked cards by default.

Show:
- one active stage expanded
- other stages collapsed/compact
- each stage has a clear status badge:
  - لم يبدأ
  - جاهز
  - مسودة
  - يحتاج مراجعة
  - مكتمل

Use cards/accordion-like layout if existing components allow it.
Do not add new libraries.

4. One primary action at a time

Avoid multiple competing generation buttons.

For the active stage, show one main CTA:
- حفظ المدخلات
- توليد موجز الاستراتيجية
- توليد اقتراحات المحتوى
- توليد المواصفات الإبداعية
- مراجعة النتائج

Do not change existing handlers.
Reuse the existing functions/mutations/buttons behind the simplified UI.
